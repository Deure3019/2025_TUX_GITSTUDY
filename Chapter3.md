## **[1] 브랜치란**

### **브랜치가 필요한 이유**
브랜치(branch)는 나뭇가지를 뜻한다.
나뭇가지가 무엇인지 모를 수 있기에 설명하자면
나뭇가지란 나무의 줄기에서 갈라져 나온 부분을 말한다.

깃에서는 브랜치는 프로그램 소스 버전을 관리하기 위한 개념이다.
버전이 반영되지 않거나 수정한 내용이 의도치 않게 바뀌거나 사라지는 문제가 발생할 수 있습니다.
이 문제들을 해결해주는 도구가 바로 브랜치입니다.

### **브랜치 기능 살펴보기**
깃으로 버전 관리를 시작하면 기본적으로 main 브랜치가 만들어진다.
사용자가 커밋할 때마다 main 브랜치는 어떤 게 최신 커밋인지 정보를 가진다.


## **[2] 브랜치 만들기 및 이동하기**

### **새 브랜치 만들기**
git branch 명령을 통해 깃에서 브랜치를 만들거나 확인할 수 있다.


## **[3] 브랜치 정보 확인하기**

### **브랜치와 커밋 관계 알아보기**
git log 명령을 사용할 때 --branches 옵션을 추가하면 브랜치마다 최신 커밋을 한눈에 볼 수 있다.
--graph 옵션을 추가하면 보기 쉽게 그래프 형태로 표시된다.


## **[4] 브랜치 병합하기**

### **서로 다른 파일 병합하기**
브랜치를 병합할 때는 git merge 명령 뒤에 가져올 브랜치 이름을 적습니다.
